# 영주선비AI 3분 데모 시나리오

> 2026 영주시 공공데이터 활용 창업경진대회 — 심사위원 시연용
> 라이브 URL: <https://yeongju-sunbi-ai.vercel.app/>

목표: 3분 안에 심사 4축(공공데이터 활용도·콘텐츠 완성도·비즈니스 가치·기술적 완성도)을 모두 시각화한다. 클릭마다 새로운 임팩트 모먼트가 나오도록 동선을 설계했다.

## 동선 한눈에

```
0:00 ─ 홈                  ─ "2026 출품작" 배지 노출, 유네스코 2건·5개 기관 stats
0:25 ─ /courses           ─ 큐레이션 3 코스 카드(당일·1박2일·2박3일) 모달 1회
0:55 ─ /chat              ─ 캐논 질문 "배흘림기둥" 클릭 → 0.5초 내 응답
1:25 ─ /heritage          ─ 선비 계보 타임라인 호버 + 네트워크 그래프 토글
2:00 ─ /heritage/buseoksa ─ 무량수전·국보·유네스코 메타 / KakaoTalk 공유 시연
2:25 ─ /stamp-tour        ─ GPS 거부 → 체험 모드 모달 → 스탬프 1개 수집
2:50 ─ /data-sources      ─ 5개 공공 API 카드 → "공공데이터 활용도" 명시
3:00 ─ 모바일 BottomNav 시연(선택)
```

---

## 0:00 — 홈 (`/`)

**시연 포인트:** 첫 인상 + 출품작 인증

- 헤로 상단 골드 배지 **"🏆 2026 영주시 공공데이터 활용 창업경진대회 출품작"** 명시
- 보조 배지 "유네스코 세계유산 2개 보유 도시 - 영주"
- 통계 카드: 2개·5개소·5개 (유네스코·스탬프·공공데이터)
- 클릭: 헤더 **"추천 코스"** 메뉴

**멘트 예시:**
> "영주선비AI는 영주시의 공공데이터를 활용한 AI 문화유산 해설 서비스로, 이미 라이브로 운영 중입니다."

## 0:25 — 추천 코스 (`/courses`)

**시연 포인트:** 콘텐츠 큐레이션 + 데이터 융합

- 3 코스 카드 노출: 당일치기 유네스코 · 1박2일 선비정신 · 가족나들이 2박3일
- "1박2일 선비정신" 카드 클릭 → 모달에 사이트 4곳(소수서원·선비촌·무섬·영주향교), 추천 시즌, 교통편 노출
- AI 챗봇의 `planTourCourse` 도구가 동일 데이터를 큐레이션 우선 반환한다는 점을 멘트

**멘트:**
> "AI 챗에 코스를 물어보면 큐레이션된 코스를 즉시 응답하고, 그 외 요청만 LLM이 새로 구성합니다."

## 0:55 — AI 해설사 (`/chat`)

**시연 포인트:** 7개 도구 + 캐논 캐시(데모 안정성)

- 추천 질문 칩 **"부석사 무량수전의 배흘림기둥은 왜 특별한가요?"** 클릭
- **0.5초 이내** 응답 시작 (캐논 Q&A 캐시 동작)
- 응답에 "그리스 파르테논 신전" 비교 등 학술 톤 노출
- 응답 우측 상단 복사 버튼 클릭 → ✓ 표시
- 헤더 "공유" 버튼 클릭 → 카카오톡 공유 인텐트 (모바일) 또는 URL 복사 (데스크탑)

**멘트:**
> "5개 핵심 질문은 사전 캐싱되어 외부 API가 늦더라도 시연 중 끊김이 없습니다."

## 1:25 — 문화유산 + 선비 계보 (`/heritage`)

**시연 포인트:** 8개 사이트 + 인물·유산 관계 시각화 ★ **가장 큰 와우 모먼트**

- 페이지 진입 시 **"영주 선비 계보 — 인물과 유산의 천 년"** 섹션이 그리드 위에 노출
- 타임라인 보기 → 의상대사(625) ~ 박수(1712) 7인의 생몰 lifespan + 8개 유산 마커
- **이황** 행 호버 → 소수서원·선비촌·소수박물관 마커 강조
- "네트워크 보기" 토글 → 인물-유산 관계 SVG 그래프 노출
- 검색 바 "부석사" 입력 → 카드 필터링

**멘트:**
> "안향에서 주세붕, 퇴계 이황까지 영주 선비 학맥을 인물-유산 관계로 시각화했습니다."

## 2:00 — 문화유산 상세 (`/heritage/buseoksa`)

**시연 포인트:** 데이터 깊이 + 공유 가능성

- 부석사 상세 페이지의 히든 스토리·아키텍처 탭
- 카카오톡 디버거 또는 URL 공유 → OG 이미지(1200×630, /og/buseoksa.jpg) 프리뷰
- 메타 정보: 유네스코 등재, 무량수전 국보 제18호, 운영 시간·요금

**멘트:**
> "각 유산 상세 페이지는 카카오톡·트위터 OG 이미지를 자동 생성합니다. (이미지 자산은 영주시 공식 사진 또는 자체 일러스트로 확장 예정)"

## 2:25 — 스탬프투어 GPS (`/stamp-tour`)

**시연 포인트:** 실전 운용 가능성 + 데모 안정성

- 진입 시 위치 권한 다이얼로그 → "차단"
- 즉시 **"🧪 체험 모드 — 영주 현장에서는 GPS 100m 반경으로 자동 검증"** 모달 노출
- 모달 한 탭으로 닫은 후, 스탬프 1개 수집 가능 (체험 모드)
- "GPS 모드로 전환" 토글 가능 (영주 현장 방문 시)
- localStorage에 진행 상황 영구 저장

**멘트:**
> "영주 현장에서는 Haversine 거리 100m로 GPS 자동 검증, 시연 환경에서는 체험 모드로 즉시 진행됩니다. 모든 권한 거부 경로가 fallback 처리되어 데모가 멈추지 않습니다."

## 2:50 — 공공데이터 출처 (`/data-sources`)

**시연 포인트:** **공공데이터 활용도 #1** 심사 축 정면 어필

- 5개 공공 API 카드:
  1. 문화재청 국가문화유산포털 → `searchHeritage`
  2. 한국관광공사 TourAPI 4.0 → `searchTourSpots`, `planTourCourse`
  3. 기상청 단기예보 API → `getWeather`
  4. 국립중앙박물관 e-Museum → `searchMuseum`
  5. 한국학중앙연구원 한국민족문화대백과사전 → `searchEncyclopedia`
- 각 카드에 endpoint URL, 활용 방식, 샘플 쿼리, REST/GET/Static 배지
- 푸터에 "공공데이터" 링크가 명시되어 누구나 1클릭으로 접근

**멘트:**
> "5개 정부·공공기관 데이터를 이렇게 활용하고 있다는 사실을 단일 페이지로 정리했습니다."

## 3:00 — (선택) 모바일 시연

심사위원 폰을 들고 BottomNav (홈/문화재/챗/퀴즈/스탬프) 5탭으로 전환 시연. Instagram/Naver 앱과 동일 UX 컨벤션.
